# PatchScope v0.1.0

PatchScope's first public release provides a local-first workbench for evidence-backed code review
and conservative refactor previews. The default workflow requires no AI provider credential and
never executes or modifies submitted code.

## Highlights

- Review pasted code, a source file, a bounded ZIP archive, or a public GitHub pull request.
- Find bugs, security risks, performance problems, and readability issues with source-linked
  evidence and analyzer provenance.
- Preview refactors as diffs, record triage decisions, and export Markdown or SARIF.
- Use deterministic analysis offline; opt into OpenAI synthesis or Semgrep separately.
- Run the FastAPI service and Streamlit workbench together with one command.

## Install and try it

Requirements: Python 3.12 and [uv](https://docs.astral.sh/uv/).

```bash
git clone https://github.com/ownasquare/patchscope.git
cd patchscope
git checkout v0.1.0
uv sync
uv run patchscope start
```

Open `http://127.0.0.1:8501`, choose **Load example review**, then **Run review**. See the
[README](https://github.com/ownasquare/patchscope/blob/v0.1.0/README.md) for the standard virtual
environment and Docker paths.

## Validation

The release candidate passed the hosted verification, container, and Playwright jobs. The release
gate covers Ruff, strict mypy, offline tests with branch coverage, Bandit, dependency audit, lock
validation, wheel and source builds, and an isolated wheel launcher smoke test.

## Supported boundary

The included server and Compose stack are local, single-user references, not hardened public
multi-user hosting. Private-repository review is outside the supported `0.1.x` contract. Refactors
are previews only, and provider-backed synthesis remains optional. PatchScope `v0.1.0` is released
on GitHub and is not published to PyPI.

Read the [security design](https://github.com/ownasquare/patchscope/blob/v0.1.0/docs/security.md)
before processing sensitive code or adapting PatchScope for shared infrastructure.
