# PatchScope v0.1.1

PatchScope `v0.1.1` hardens the optional GitHub and OpenAI paths while keeping the offline first run
unchanged. It is a focused trust-boundary and contributor-experience release.

## Highlights

- Reject private or ambiguous GitHub repository metadata before requesting files or source.
- Cap the complete OpenAI prompt and completion, with persisted per-section usage and truncation.
- Preserve every deterministic finding when provider context is shortened or synthesis falls back.
- Disable LangSmith tracing around reviews and pass only recognized settings to child processes.
- Block TCP access in default tests while keeping live and browser suites explicitly opt-in.
- Document the optional PostgreSQL driver boundary and remove unused dependency layers.

## Install and try it

Requirements: Python 3.12 and [uv](https://docs.astral.sh/uv/).

```bash
git clone https://github.com/ownasquare/patchscope.git
cd patchscope
git checkout v0.1.1
uv sync
uv run patchscope start
```

Open `http://127.0.0.1:8501`, choose **Load example review**, then **Run review**. The default path
is local and credential-free.

## Validation

The release gate covers formatting and lint, strict typing, offline tests with branch coverage,
Bandit, dependency audit, lock validation, source and wheel builds, and an isolated wheel launcher
smoke test. Playwright separately validates the combined API/workbench flow. Authenticated public
GitHub intake passed its bounded live fixture; OpenAI live acceptance remains unclaimed until a
maintainer supplies a local API key.

## Supported boundary

The included server and Compose stack remain local, single-user references. Private-repository
review, hosted multi-user hardening, PostgreSQL migration lifecycle, and automatic refactor writes
are outside the supported `0.1.x` contract.

Read the [security design](https://github.com/ownasquare/patchscope/blob/v0.1.1/docs/security.md)
before processing sensitive code or adapting PatchScope for shared infrastructure.
